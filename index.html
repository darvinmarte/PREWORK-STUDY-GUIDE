HTML

  • Enter your HTML notes here
  • The head element contains information about the webpage
  • The head element is a container for metadata
  • Meta elements contain information about the page that is used by the browser
  • The user wont see information in the head or meta elements.
  • Body element contains the content that is shown to the user. All the elements that we want the user to see and interact with must be placed inside this element.
  • H1, h2, h3, h4, h5, and h6 elements represent the level of heading a given text block represents
  • P element represents a paragraph or block of text
  • Ul, ol, and li elements represent unordered lists, ordered lists, and list items
  • Img element contains information about images that are displayed on the webpage, src attribute is the most important because it defines the location of the image
  • `Alt` attribute contains a text string that describes the appearance and functionality of the image.
  • Br element creates a line of empty space, or a line break, between two blocks of content
  • A element (which stands for anchor) creates links to the same webpage or other webpages, href attribute points to the URL for the link
  • Footer appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation.

CSS

  • Enter your CSS notes here
  • Cascading Style Sheets (CSS), to give the content style, programming language used by developers to define how webpage content is presented to users
  • inline CSS- style an element directly in the HTML file, useful for quick and permanent changes but are less flexible than external and internal style sheets, because each inline style must be edited individually whenever you make a design change.
  • internal style sheet- contains CSS rules for the webpage in the head section of the HTML file.
  • External style sheet: most common way to style a webpage, allows developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • selector defines the element or attributes to which the rules, or declarations, will apply
  • Declarations contain two important components: the CSS property we want to apply and the value of the property. For EX, the declaration color: blue; includes the CSS property color and the value for that property, blue.
  • margin property: indicates how much space we want around the outside of an element
  • *- applies a rule to all the elements visible on the page EX: we have declared that ALL of the elements on the page will have the style properties margin and padding set as 0. * {margin: 0; padding: 0;}
  • selector assigns property values to any image element in the markup.
  • display property is assigned with the block value, which assigns each image to start on a new line.
  • element: creates the connection to the external file. when we create an external CSS file, we need to link it to the HTML file. We store metadata, which contains information about the webpage. This is where we will add the link to our style sheet
  • rel attribute specifies the relationship between the current document and the linked document resource.
  • href attribute specifies the location (URL) of the external resource.
  • box-shadow CSS property adds shadow effects around an element's frame. You can set multiple effects separated by commas. A box shadow is described by X and Y offsets relative to the element, blur and spread radius, and color. In our example, the X offset is 5px and defines the width or extent of the shadow on the X axis (or to the right of the box); the Y offset is 10px and defines the extent on the Y axis (or just below the box). We haven’t defined the blur or spread radius here, but you’ll learn more about this in time. We’ve used an HTML hexadecimal code to define the color.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
